# Smart Queue Prediction and Token Sharing System

## Features
- **Complete Authentication:** Role-based access (User/Admin).
- **Token Booking:** Book tokens across multiple queues.
- **Wait Time Prediction:** Live estimated wait times using a **Random Forest Machine Learning Algorithm** to dynamically predict service times based on historical data.
- **Token Sharing:** Request a token transfer to another user.
- **Admin Panel:** Create queues, review transfer requests, and call next tokens.
- **Real-time WebSockets:** Live queue updates broadcasted to all users via `Socket.IO`.

## Tech Stack
- **Backend:** Python, Flask, Flask-SQLAlchemy.
- **Machine Learning:** Scikit-Learn (Random Forest Regressor), NumPy.
- **Real-Time & Auth:** Flask-SocketIO, Flask-JWT-Extended.
- **Database:** PostgreSQL.
- **Frontend:** HTML, CSS, Vanilla JavaScript (Axios, Socket.IO client).

## 🚀 Getting Started

### 1. Database Setup
1. Open PostgreSQL (via psql or pgAdmin) and create the database:
   ```sql
   CREATE DATABASE smart_queue_db;
   ```
2. Note: The database schema will be automatically generated by SQLAlchemy when you run the backend server (`db.create_all()` in `app.py`).

### 2. Backend Setup
1. Navigate to the `backend` folder:
   ```bash
   cd backend
   ```
2. Set up a virtual environment (optional but recommended):
   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # macOS/Linux
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Configure `.env` variables:
   Copy `.env.example` to `.env`. Update the `DATABASE_URL` if your PostgreSQL username/password is different.
5. Run the server:
   ```bash
   python app.py
   ```
   > The Flask API with WebSockets will run on port `5000`.

### 3. Frontend Setup
1. Since the frontend is written in vanilla HTML/CSS/JS, no build tools are required.
2. Open the `frontend` directory.
3. Serve it using any static server to avoid CORS/file protocol restrictions:
   ```bash
   # Using npx (Node)
   npx serve .
   
   # Or using Python 3
   python -m http.server 8000
   ```
4. Navigate to `http://localhost:8000` in your web browser.

### 4. Admin Access
For testing purposes, the registration page includes an option to register as an `Admin`. Select it when signing up to access the Admin Panel, where you can create queues and manage token transfers.
